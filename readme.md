# 伴伴智宠 - Ban-Ban Dog 使用说明

## 版本信息
- **版本**：V2.1

- **更新时间**：2026-03-17

## 功能概述
伴伴智宠是一款基于ESP32的四足机器人，支持多种控制模式：
- 蓝牙控制模式（微信小程序）
- 网页控制模式（WiFi热点）
- 串口控制模式（兼容JSON及HEX指令，支持语音模块）

## 版本更新日志

### V2.1 更新（2026-03-17）

- **微信小程序语音模式兼容**：蓝牙控制模式下，串口语音模块可同时工作，实现微信小程序控制与语音控制的无缝切换
- **双模式协同优化**：蓝牙BLE指令和串口语音指令使用统一解析器，确保两种控制方式互不干扰，响应更及时
- **代码结构优化**：重构主程序架构，分离蓝牙服务和运动控制逻辑，提高代码可维护性
- **启动流程优化**：改进双启动模式检测机制，提升模式切换稳定性
- **指令系统增强**：统一指令解析器，更好支持 JSON 和 HEX 格式指令
- **摇杆控制优化**：改进蓝牙摇杆数据处理，支持更平滑的全向移动控制
- **系统稳定性提升**：优化定时器和线程管理，减少运行时的资源冲突

### V2.0 更新（双启动蓝牙版）

- **双启动模式 (Dual Boot)**：解决 WiFi 与 蓝牙 资源冲突问题，支持用户模式（蓝牙）和调试模式（WiFi热点）两种启动方式
- **用户模式优化**：默认开机进入蓝牙模式，板载 LED 常亮，蜂鸣器提示，支持微信小程序连接控制
- **调试模式增强**：开机按住 BOOT 键进入 WiFi 热点模式（热点名: Ban-Ban Dog - 调试模式），支持网页标定
- **微信小程序控制升级**：支持蓝牙 BLE 连接（设备名: BanBan-Dog），新增全向摇杆控制，支持平滑变速与转向
- **动作指令兼容**：完整支持 V1.4 所有动作指令（跳舞、伸懒腰、俯卧撑等）
- **底层驱动重构**：适配 MicroPython v1.20+ 新版固件，修复 PWM 频率报错和 SoftI2C 兼容性问题
- **电压检测优化**：重写电压检测算法，校准 ADC 系数，修复 USB 供电误报亏电导致舵机锁死的问题
- **指令解析器改进**：优化指令解析器，解决蓝牙高速传输时的"粘包"问题，摇杆控制更丝滑

**使用说明**：
- **用户模式（手机玩耍）**：开机不按任何键，等待蜂鸣器提示后，通过微信小程序"伴伴智宠"连接 BanBan-Dog 设备
- **调试模式（网页标定）**：开机按住 BOOT 键，连接 WiFi 热点"Ban-Ban Dog - 调试模式"，访问 192.168.4.1 进入控制后台

**注意事项**：
- **电压报警**：机器人急促报警且舵机无力时，说明电池电压过低，请及时充电
- **USB调试**：V2.0+ 版本开启 DEBUG_USB_MODE，允许仅连接 USB 时驱动舵机（仅限调试小动作），正式玩耍时请连接动力电池
- **舵机发热**：长时间运行"自稳模式"或"高频步态"会导致舵机发热，建议玩耍 10 分钟休息片刻

### V1.4 更新（2025-10-14）

- **姿态切换系统全面优化**：实现S曲线速度控制，采用6x⁵-15x⁴+10x³数学模型提供平滑加减速，彻底消除姿态切换时的冲击现象。
- **动态起始位置检测**：所有姿态（站立/坐下/趴下）都使用真实当前动态参数计算起始位置，避免理论位置与实际位置的偏差。
- **姿态参数平滑过渡**：采用渐进式姿态调整策略，姿态角逐步调整（PIT_S×0.3→0.2→0.1）而非直接跳变到目标值。
- **切换速度大幅提升**：优化过渡参数，姿态切换时间从1.5秒缩短到0.375-0.7秒，同时保持动作流畅性。
- **修复所有姿态切换冲击**：解决坐下→站立/趴下、趴下→坐下/站立时的"加速顿挫"问题，实现所有姿态间切换的完全流畅。

### V1.3 更新（2025-09-04）

- 新增HEX指令兼容：增加了对固定16进制串口指令的支持，方便与其他硬件模块对接。
- 修复动作中断问题：修复了打招呼、坐下等长时动作会被主循环打断的问题，确保动作可以完整执行。
- 优化站立姿态：彻底解决了stand指令与用户设定的重心不一致的问题，站立姿态更稳定。

### V1.2 更新（2025-09-02）
- 运动平顺性优化：修复了在运动中切换方向（如前进时左转）导致的卡顿问题，运动过渡更自然。
- 转弯指令简化：left和right指令修改为与forward一致的3秒限时动作，无需额外参数。
- 舞蹈功能简化：移除了多个舞蹈动作，仅保留一个经典舞蹈动作，指令简化为dance。

### V1.1 更新（2025-08-28）
- 串口模式简化：forward和backward指令现在直接使用3秒固定时间运动，无需速度参数。
- 默认最大速度：基础运动指令默认使用最大速度。
- 后退优化：修复后退时腿部过低的问题，调整重心参数。

### V1.0 更新（Web端与语音控制增强版）

- **Web端控制界面优化**：重构控制页面逻辑，新增语音模式切换按钮，支持一键切换至语音控制模式
- **新增语音模式页面**：独立语音控制界面（voice_mode.html），便于语音模块调试和状态显示
- **串口控制增强**：新增语音模式串口指令支持（HEX格式），可通过语音模块发送特定16进制指令控制机器人动作
- **控制模式切换**：支持网页模式（CC_M=0）、航模遥控模式（CC_M=1）、串口/语音模式（CC_M=2）三种模式灵活切换

## 串口控制指令汇总 (支持JSON和HEX两种格式)
### 1. JSON格式指令 (推荐)
- `{"DJCtrl":"forward"}` - 前进3秒
- `{"DJCtrl":"backward"}` - 后退3秒
- `{"DJCtrl":"left"}` - 左转3秒
- `{"DJCtrl":"right"}` - 右转3秒
- `{"DJCtrl":"stop"}` - 停止所有动作
- `{"DJCtrl":"stand"}` - 站立
- `{"DJCtrl":"sit"}` - 坐下（并保持）
- `{"DJCtrl":"lie"}` - 趴下（并保持）
- `{"DJCtrl":"greet"}` - 打招呼动作
- `{"DJCtrl":"dance"}` - 舞蹈
- `{"DJCtrl":"stable_on"}` - 开启自稳
- `{"DJCtrl":"stable_off"}` - 关闭自稳
- `{"DJCtrl":"gait_trot"}` - 设置小跑步态
- `{"DJCtrl":"gait_walk"}` - 设置慢走步态
- `{"DJCtrl":"mode_web"}` - 切换到网页控制模式

### 2. 16进制格式指令 (兼容)
- 前进: FF 01 EF
- 后退: FF 02 EF
- 左转: FF 03 EF
- 右转: FF 04 EF
- 停止: FF 05 EF
- 站立: FF 06 EF
- 坐下: FF 07 EF
- 趴下: FF 08 EF
- 伸懒腰: FF 11 EF
- 抖身体: FF 12 EF
- 舞蹈: FF 13 EF
- 俯卧撑: FF 14 EF
- 网页模式: FF 10 EF

## 接线说明

### 串口控制模式
- TX: 引脚26
- RX: 引脚27
- GND: 舵机接口处黑色针脚

## 使用注意事项

- **双启动模式**：V2.0+ 版本支持双启动模式，开机不按按键进入蓝牙模式，按住 BOOT 键开机进入 WiFi 调试模式
- **微信小程序与语音模式共存**：V2.1+ 版本支持蓝牙控制时同时使用语音模块控制，通过串口连接语音模块即可实现语音指令控制
- **姿态切换优化**：V1.4版本全面优化了姿态切换系统，所有姿态间切换（站立↔坐下↔趴下）均采用S曲线速度控制，动作流畅无冲击，切换速度提升60%。
- **限时动作**：forward、backward、left、right等指令均为阻塞式执行，执行期间不响应其他指令。
- **姿态保持**：lie和sit指令会让机器人持续保持姿态。机器人不会自动恢复站立，必须通过发送新的移动指令（如forward、stand等）来使其恢复运动。
- **动作保护**：打招呼、坐下、趴下、舞蹈等特殊动作在执行时会自动暂停底层姿态循环，以防止动作被中断。
- **串口格式**：系统优先尝试以JSON格式解析串口数据。如果解析失败，则会尝试匹配16进制指令。
- **安全保护**：当电压低于5.5V时，系统会发出警报并停止舵机输出以保护电池。
